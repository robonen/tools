## Как читать

- `Purpose` — зачем preset подключать.
- `Key Rules` — ключевые правила из preset-а (не полный dump).
- `Examples` — минимальные `good/bad` примеры.

Для точного источника правил см. файлы в `configs/oxlint/src/presets/*.ts`.
